# ADR-001: Вибір СУБД для системи інтернет-магазину

## Статус
Accepted

## Контекст
Система обробляє ~5000 замовлень/добу, ~200 одночасних користувачів. 
Дані структуровані, потрібна підтримка транзакцій (ACID), 
можливість зберігання JSON-даних для гнучких атрибутів товарів.

## Рішення
Використовувати PostgreSQL 16 як основну реляційну СУБД.

## Обґрунтування
1. ✅ Повна підтримка ACID-транзакцій для гарантії цілісності замовлень
2. ✅ Вбудована підтримка JSONB для гнучких атрибутів без втрати продуктивності
3. ✅ Повнотекстовий пошук для каталогу товарів без додаткових інструментів
4. ✅ Безкоштовна ліцензія (PostgreSQL License) та активна спільнота

## Альтернативи
| Варіант | Чому відхилено |
|---------|---------------|
| MySQL 8.0 | Менш розвинена підтримка JSON, слабший оптимізатор запитів |
| MongoDB | Не підходить для реляційних зв'язків (замовлення ↔ товари ↔ користувачі) |
| SQLite | Не масштабується для багатокористувацького навантаження |

## Наслідки
### ✅ Позитивні
- Гарантія цілісності даних при одночасних операціях
- Можливість міграції на хмарні рішення (AWS RDS, Google Cloud SQL)

### ⚠️ Негативні
- Потрібен DBA для налаштування індексів та реплікації
- Більша вимогливість до ресурсів порівняно з SQLite
