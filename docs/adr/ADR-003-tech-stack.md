# ADR-003: Вибір технологічного стеку: Node.js + React + PostgreSQL

## Статус
Accepted

## Контекст
Проєкт інтернет-магазину потребує:
- Backend: REST API з автентифікацією (JWT), роботою з БД, інтеграцією з платіжними системами
- Frontend: інтерактивний UI з динамічним каталогом, кошиком, особистим кабінетом
- Команда знайома з JavaScript/TypeScript
- Потрібна швидка розробка та велика екосистема бібліотек
- Бюджет обмежений (потрібні безкоштовні/open-source рішення)

## Рішення
**Backend**: Node.js 20 LTS + Express.js + TypeScript  
**Frontend**: React 18 + TypeScript + Vite  
**Спільна мова**: TypeScript для повного стеку  
**Додатково**: Prisma ORM, JWT для автентифікації, Jest для тестів

## Обґрунтування
1. ✅ **Єдина мова (JavaScript/TypeScript)**: команда працює з одним синтаксисом на frontend та backend; легше ротація розробників між шарами
2. ✅ **Велика екосистема npm**: готові рішення для будь-яких задач (passport для auth, stripe для платежів, multer для файлів)
3. ✅ **Швидка розробка**: Express.js мінімалістичний, не вимагає boilerplate коду; React має велику спільноту та готові UI-бібліотеки (Material-UI, Ant Design)
4. ✅ **Type Safety**: TypeScript виявляє помилки на етапі компіляції, покращує IDE autocompletion, спрощує рефакторинг
5. ✅ **Продуктивність**: Node.js non-blocking I/O ідеальний для I/O-операций (запити до БД, API); React Virtual DOM оптимізує рендеринг
6. ✅ **Безкоштовна ліцензія**: всі технології open-source, немає витрат на ліцензування

## Альтернативи
| Варіант | Чому відхилено |
|---------|---------------|
| **Python (Django/FastAPI) + Vue.js** | Команда не має досвіду з Python; менша продуктивність при CPU-інтенсивних задачах; два різних синтаксиси (Python + JS) |
| **Java (Spring Boot) + Angular** | Надмірна складність для проєкту; повільна розробка через boilerplate код; високе споживання пам'яті; потрібен потужніший сервер |
| **PHP (Laravel) + Blade** | Застарілий підхід (server-side rendering); складніше реалізувати SPA; менша типізація порівняно з TypeScript |
| **C# (.NET Core) + Blazor** | Vendor lock-in на Microsoft екосистему; менша гнучкість frontend; команда не знає C# |

## Наслідки
### ✅ Позитивні
- Швидкий онбординг нових розробників (велика спільнота, багато туторіалів)
- Можливість використовувати isomorphic JavaScript (код, що працює і на client, і на server)
- Легкий пошук розробників на ринку праці
- Сучасний стек привабливий для розробників

### ⚠️ Негативні
- Node.js single-threaded: потрібно використовувати cluster або worker threads для CPU-інтенсивних задач
- JavaScript екосистема швидко змінюється: потрібно постійно оновлювати залежності
- Callback hell (вирішується async/await та Promise)
- TypeScript додає крок компіляції до процесу розробки
